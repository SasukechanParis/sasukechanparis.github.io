## 4. スタッフ権限システム（今回完成）

### 動作フロー
```
スタッフがログイン
  ↓
collectionGroup('staffs').where('email','==', staffEmail) でオーナーUID特定
  ↓
staffAccessContext = { isManagedStaff: true, adminUid: ownerUid, role, viewScope }
  ↓
loadSettingsForOwner(ownerUid) でオーナーの設定を読み込む（屋号・通貨等）
  ↓
fetchClientsForCurrentUser(ownerUid) でオーナーの案件を取得
  ↓
viewScope === 'own_only' → assignedStaffEmail でフィルタ
viewScope === 'team_all' → 全件表示
```

### Firestoreインデックス（作成済み）
- Collection Group: `staffs`
- Field: `email`
- Order: ASC

### 権限制御（実装済み）
| 操作 | owner | viewer staff | 未定義 |

| 案件閲覧 | ✅ | ✅ | ❌ |
| 案件編集 | ✅ | ❌ | ❌ |

## 5. Firestoreセキュリティルール（最新版）

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // オーナーは自分のデータだけ
    match /users/{uid} {
      allow read, write: if request.auth != null 
                         && request.auth.uid == uid;
    }
    match /users/{uid}/{document=**} {
      allow read, write: if request.auth != null 
                         && request.auth.uid == uid;
    }

    // スタッフ判定関数
    function isStaffOf(ownerUid) {
      let indexPath = /databases/$(database)/documents/users/$(ownerUid)/meta/staffEmails;
      return request.auth != null
        && exists(indexPath)
        && get(indexPath).data.emails.hasAny([request.auth.token.email]);
    }

    // スタッフが自分のstaffドキュメントだけ読める
    match /{path=**}/staffs/{staffDocId} {
      allow read: if request.auth != null
                  && resource.data.email == request.auth.token.email;
    }

    // オーナー設定（読み取りのみ）
    match /users/{ownerUid}/meta/state {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }

    // 案件データ（読み取りのみ）
    match /users/{ownerUid}/projects/{clientId} {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }
    match /users/{ownerUid}/clients/{clientId} {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }

    // staffEmailsはオーナーのみ読み書き可
    match /users/{ownerUid}/meta/staffEmails {
      allow read: if request.auth != null 
                  && request.auth.uid == ownerUid;
      allow write: if request.auth != null 
                   && request.auth.uid == ownerUid;
    }
  }
}
```

## 11. よくあるエラーと対処法

| エラー | 原因 | 対処 |
|--------|------|------|
| `userMetaRef(...).get is not a function` | firebase-service.jsでcompat SDKを使った | v9パターンに変更: `getDoc(doc(db,...))` |
| `Missing or insufficient permissions` | Firestoreルールがブロック | ルールを確認・staffEmailsの存在確認 |
| `collectionGroup requires index` | Firestoreインデックス未作成 | Consoleでインデックス作成 |
| `loadSettingsForOwner is not a function` | firebase-service.jsにメソッド未追加 | loadSettingsForOwnerを追加 |
| スタッフでログインしても0件 | staffEmailsが空 | オーナーでログイン→チーム管理を開く |
