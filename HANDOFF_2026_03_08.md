
## 5. Firestoreセキュリティルール（最新版）

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
    match /users/{uid}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
    function isStaffOf(ownerUid) {
      let indexPath = /databases/$(database)/documents/users/$(ownerUid)/meta/staffEmails;
      return request.auth != null
        && exists(indexPath)
        && get(indexPath).data.emails.hasAny([request.auth.token.email]);
    }
    match /{path=**}/staffs/{staffDocId} {
      allow read: if request.auth != null
                  && resource.data.email == request.auth.token.email;
    }
    match /users/{ownerUid}/meta/state {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }
    match /users/{ownerUid}/projects/{clientId} {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }
    match /users/{ownerUid}/clients/{clientId} {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }
    match /users/{ownerUid}/meta/staffEmails {
      allow read, write: if request.auth != null && request.auth.uid == ownerUid;
    }
  }
}
```

## 6. 【NEW】ストレージモード切り替え機能（未実装・P1）

### 設計方針
- **オプトイン方式**でクラウド同期を選択させる
- ユーザーが明示的に選んだ場合のみ、リスクはユーザーが負う
- 同意日時をFirestoreに記録 → 法的証拠として機能

### フロー
```
初回ログイン
  ↓
☁️ クラウド自動同期  or  📱 ローカル保存（手動）
  ↓                        ↓
同意日時をFirestoreに記録   localStorageのみ使用
スタッフ機能使用可          スタッフ機能は無効
```

### 実装設計
```javascript
const STORAGE_MODE_KEY = 'photocrm_storage_mode'; // 'cloud' | 'local'

function isCloudMode() { return localStorage.getItem(STORAGE_MODE_KEY) !== 'local'; }

// 同意記録（クラウド選択時）
await db.collection('users').doc(uid).collection('meta').doc('state')
  .update({ cloudConsentAt: new Date().toISOString() });

// 抽象レイヤー（段階的に差し替え）
async function saveClient(data) {
  return isCloudMode() ? saveClientToFirestore(data) : saveClientToLocalStorage(data);
}
async function fetchClients() {
  return isCloudMode() ? fetchClientsFromFirestore() : fetchClientsFromLocalStorage();
}

// クラウド→ローカル移行
async function migrateCloudToLocal() {
  const clients = await fetchClientsFromFirestore();
  localStorage.setItem('photocrm_clients', JSON.stringify(clients));
  localStorage.setItem(STORAGE_MODE_KEY, 'local');
  // Firestoreのデータは削除しない（戻せるように保持）
}
```

## 7. 【NEW】言語リリースフラグ機能（未実装・P1）

### 設計方針
- 翻訳は全言語すでに実装済み（locales.jsに存在）
- **Firebase Consoleを1行変えるだけ**で言語を解放できる
- コードのデプロイ不要・即時反映

### 段階的解放ロードマップ
```
今           → ja のみ
英語圏解放    → ja, en
韓国展開      → ja, en, ko
台湾展開      → ja, en, ko, zh-TW
EU正式展開    → ja, en, ko, zh-TW, fr  ← 弁護士レビュー後
中国          → 別途判断（法規制が複雑）
```

### 実装設計
```javascript
// admin_config/releaseFlags ドキュメントに保存
// Firebase Console から直接編集する
{
  "releasedLanguages": ["ja"]  // ← ここを変えるだけで解放
}

// app.js 起動時に取得
async function fetchReleasedLanguages() {
  const snap = await db.collection('admin_config').doc('releaseFlags').get();
  return snap.exists ? snap.data().releasedLanguages : ['ja'];
}

// 言語セレクターに適用
function renderLanguageSelector(releasedLangs) {
  const ALL_LANGS = ['ja', 'en', 'fr', 'zh-CN', 'zh-TW', 'ko'];
  return ALL_LANGS.filter(l => releasedLangs.includes(l));
  // 未解放言語はセレクターに表示しない
}

// EUユーザー検知バナー（EU向け広告を出していない証拠として機能）
const euLangs = ['de','fr','it','nl','pl','pt','sv','fi','da','el'];
if (euLangs.some(l => navigator.language.startsWith(l)) && !releasedLangs.includes('fr')) {
  showBanner('このサービスはEU圏での正式提供を準備中です。');
}
```

### Firebase Consoleでの操作手順
```
Firestore → admin_config → releaseFlags
→ releasedLanguages 配列に "en" を追加
→ 即時反映（デプロイ不要）
```
