## 4. スタッフ権限システム（完成済み）

### 動作フロー
```
スタッフがログイン
  ↓
collectionGroup('staffs').where('email','==', staffEmail) でオーナーUID特定
  ↓
staffAccessContext = { isManagedStaff: true, adminUid: ownerUid, role, viewScope }
  ↓
loadSettingsForOwner(ownerUid) でオーナーの設定を読み込む（屋号・通貨等）
  ↓
fetchClientsForCurrentUser(ownerUid) でオーナーの案件を取得
  ↓
viewScope === 'own_only' → assignedStaffEmail でフィルタ
viewScope === 'team_all' → 全件表示
```

### Firestoreインデックス（作成済み）
- Collection Group: `staffs`
- Field: `email`
- Order: ASC

## 5. Firestoreセキュリティルール（最新版）

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // オーナーは自分のデータだけ
    match /users/{uid} {
      allow read, write: if request.auth != null 
                         && request.auth.uid == uid;
    }
    match /users/{uid}/{document=**} {
      allow read, write: if request.auth != null 
                         && request.auth.uid == uid;
    }

    // スタッフ判定関数
    function isStaffOf(ownerUid) {
      let indexPath = /databases/$(database)/documents/users/$(ownerUid)/meta/staffEmails;
      return request.auth != null
        && exists(indexPath)
        && get(indexPath).data.emails.hasAny([request.auth.token.email]);
    }

    // スタッフが自分のstaffドキュメントだけ読める
    match /{path=**}/staffs/{staffDocId} {
      allow read: if request.auth != null
                  && resource.data.email == request.auth.token.email;
    }

    // オーナー設定（読み取りのみ）
    match /users/{ownerUid}/meta/state {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }

    // 案件データ（読み取りのみ）
    match /users/{ownerUid}/projects/{clientId} {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }
    match /users/{ownerUid}/clients/{clientId} {
      allow read: if isStaffOf(ownerUid);
      allow write: if false;
    }

    // staffEmailsはオーナーのみ読み書き可
    match /users/{ownerUid}/meta/staffEmails {
      allow read: if request.auth != null 
                  && request.auth.uid == ownerUid;
      allow write: if request.auth != null 
                   && request.auth.uid == ownerUid;
    }
  }
}
```

## 6. 【NEW】ストレージモード切り替え機能（未実装・次回実装）

### 背景・設計判断
- クラウド同期は便利だが、プライバシーを重視するユーザーへの配慮が必要
- 各国のデータ保護法（GDPR等）への対応として「オプトイン方式」を採用
- **オプトイン = ユーザーが明示的に選んだ場合のみ、責任はユーザーが負う**
- 現時点で既存ユーザーはいないため、アーキテクチャ変更のリスクなし

### ユーザー体験フロー
```
初回ログイン or 設定画面
  ↓
┌──────────────────────────────────────┐
│  データの保存方法を選んでください      │
│                                      │
│  ☁️ クラウド自動同期                  │
│  　デバイス間で自動同期。              │
│  　データはFirebaseに保存されます。    │
│  　万が一の漏洩リスクはご自身で        │
│  　ご負担ください。        [選択する]  │
│                                      │
│  📱 ローカル保存（手動同期）           │
│  　データはこのデバイスにのみ保存。    │
│  　エクスポート/インポートで           │
│  　手動同期できます。       [選択する]  │
└──────────────────────────────────────┘
       ↓                    ↓
STORAGE_MODE='cloud'   STORAGE_MODE='local'
Firestoreに保存        localStorageのみ
いつでも切替可         設定から変更可能
```

### 実装設計

#### ① ストレージモードの管理（app.jsに追加）
```javascript
const STORAGE_MODE_KEY = 'photocrm_storage_mode'; // 'cloud' | 'local'
const CLOUD_CONSENT_KEY = 'photocrm_cloud_consent'; // 同意日時

function getStorageMode() {
  return localStorage.getItem(STORAGE_MODE_KEY) || 'cloud'; // 既存互換でデフォルトcloud
}
function isCloudMode() { return getStorageMode() === 'cloud'; }
function isLocalMode() { return getStorageMode() === 'local'; }
```

#### ② データ読み書きの抽象化（段階的に差し替える）
```javascript
// 現在: Firestoreに直接書き込み
// 変更後: モードで振り分ける抽象レイヤーを追加

async function saveClient(clientData) {
  if (isCloudMode()) {
    return await saveClientToFirestore(clientData); // 既存処理
  } else {
    return saveClientToLocalStorage(clientData);    // 新規
  }
}

async function fetchClients() {
  if (isCloudMode()) {
    return await fetchClientsFromFirestore();       // 既存処理
  } else {
    return fetchClientsFromLocalStorage();          // 新規
  }
}
```

#### ③ モード切り替え時のデータ移行
```javascript
// クラウド → ローカル
async function migrateCloudToLocal() {
  const allClients = await fetchClientsFromFirestore();
  localStorage.setItem('photocrm_clients', JSON.stringify(allClients));
  localStorage.setItem(STORAGE_MODE_KEY, 'local');
  // Firestoreのデータは削除しない（戻せるように保持）
}

// ローカル → クラウド（同意確認後）
async function migrateLocalToCloud() {
  const clients = JSON.parse(localStorage.getItem('photocrm_clients') || '[]');
  for (const client of clients) {
    await saveClientToFirestore(client);
  }
  localStorage.setItem(STORAGE_MODE_KEY, 'cloud');
  localStorage.setItem(CLOUD_CONSENT_KEY, new Date().toISOString());
}
```

### 実装上の難所と対策

| 難所 | 対策 |
|------|------|
| app.jsが11,800行・Firestore操作が無数 | 抽象レイヤー関数を先に作り、段階的に差し替え |
| 一気に変更するとバグリスク大 | まずsaveClient/fetchClientsの2関数だけ対応し動作確認 |
| スタッフ機能はFirestore前提 | ローカルモードではチーム機能を無効化し明示 |
